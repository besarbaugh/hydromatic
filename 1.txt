Modernize the Workload Identity Access Management (WIAM) platform, extend it to GCP, and prototype a unified cross-cloud access management tool. Evaluate integrating identity entitlements into the landing zone framework so workload access is provisioned declaratively alongside infrastructure rather than requested after the fact.

Key Deliverables
- Complete WIAM on-prem migration from Flask monolith to FastAPI containerized architecture (carryover from 2025; test automation complete).
- Extend WIAM self-service RBAC capabilities to GCP resources and projects using patterns proven in Azure.
- POC a unified cross-cloud tool providing a single interface for workload identity RBAC across Azure and GCP.
- Evaluate shifting CIAM from reactive access granting to declarative identity governance: assess embedding approved identity entitlements into landing zone Terraform modules with policy-as-code guardrails, so entitlements deploy with infrastructure. Document recommendation including WIAM's evolved role as exception handler and audit surface.

Success Criteria
- WIAM on-prem migration complete (FastAPI containerized) by early Q2 2026.
- GCP WIAM extension functional in non-prod by Q3 2026.
- Unified cross-cloud tool POC demonstrated to CIAM leadership by Q4 2026.
- Landing zone identity integration evaluation complete with recommendation by Q4 2026.

Timeline
Apr-May: WIAM on-prem migration wrapped up and signed off.
Q3: GCP WIAM extension in non-prod. Unified tool POC underway. Begin landing zone evaluation.
Q4: Unified tool POC demonstrated. Operating model recommendation presented to leadership.

Business Value
Modernizes CIAM's core platform while expanding to GCP. Evaluating declarative identity entitlements within landing zones positions CIAM to shift from access bottleneck to governance platform, enabling true infrastructure-on-demand where identity is provisioned alongside the resources it protects.
